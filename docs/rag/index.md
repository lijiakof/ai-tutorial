# 如何实现 RAG

## 什么是 RAG
RAG（检索增强生成） 是一种让大语言模型（LLM）“先查资料，再回答问题” 的技术框架。它的核心思路是：在模型生成答案之前，先从外部知识库（如文档、数据库、网页）中检索出最相关的信息，把这些信息作为上下文拼接进提示词，再交给模型生成最终回答。

### 为什么需要 RAG
纯 LLM 有三大痛点：

- 📅 知识截止：只能回答训练数据覆盖的内容，无法获取最新信息。
- 🤥 幻觉问题：遇到不懂的问题可能“一本正经地胡说八道”。
- 🔒 隐私与领域知识：无法访问企业内部文档、私有数据。

RAG 把这些痛点一揽子解决：用外部知识弥补模型内部知识的不足，让回答有据可查、时效性强、可溯源。

如何解决：将所有的文档（知识库）给 LLM 处理，带来如下问题：
- 上下文大小限制
- 计算成本高
- 响应速度慢

## RAG 核心流程

在模型回答前，先检索资料，再基于资料生成答案。

### 第一阶段：数据准备

- 知识库切分（chunking）
	- 切分原则：每一块语义相对完整
	- 切分方式：章节、段落、字数等
- 向量化（Embeding）
	- 将块（chunk）按照坐标计算出位置
	- 存入向量数据库
		- 块->坐标

### 第二阶段：用户提问
- 问题向量化
- 问题优化
- 相似度计算
	- 余弦相似度
	- 欧式相似度（距离）
- Top-K 召回
- 重排序（rerank）
- LLM 处理：提问+rerank+提示词给 
- 返回问题

## 优化技巧
- 数据清洗
	- PDF、表格、图片
- 块切分
	- 太大：导致内容太多，检索不精确
	- 太小：造成语义断裂
- 用户提问
	- 问题重写，优化问题
- 问题检索
	- 混合检索：关键词+向量相似度检索
- 模型：
	- 稳定
	- 遵循规则
	- 开源
